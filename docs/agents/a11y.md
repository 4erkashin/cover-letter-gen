# Accessibility

Living cheat sheet of **a11y patterns this repo already ships**. Append when you add a new pattern. Do **not** turn this into a WCAG course or invent practices not present in code.

Real product trade-offs (competing options you chose) still belong in `docs/adr/`, not here.

## Patterns in use

**Named graphic, hide the parts** — Parent gets `role="img"` + `aria-label`. Decorative children get `aria-hidden` so AT hears one name, not empty nodes.

- Goal progress: `features/goal/progress.tsx` + `features/goal/indicator.tsx`
- App logo: `ui/app-logo/app-logo.tsx`
- Goal-reached check: `features/goal/status.tsx`
- Generating preloader: `features/letter-form/letter-preview/letter-preview.tsx`

**Icon-only / logo control name** — Visible label absent → put the accessible name on the control (`aria-label` on `Link` / labeled `role="img"`).

- Home control: `app/layout.tsx`
- Logo mark: `ui/app-logo/app-logo.tsx`

**Busy region** — While content is loading/generating, mark the region `aria-busy` and keep a stable `aria-label`.

- Letter preview: `features/letter-form/letter-preview/letter-preview.tsx`
- Goal status skeleton: `features/goal/status.tsx`

**Status announcements** — Generation start/success/failure go through a polite live region (`aria-live="polite"` + `role="status"`), visually hidden when copy is for AT only.

- Letter form: `features/letter-form/letter-form.tsx`

**Control name while chrome changes** — When a button label is replaced by a spinner, keep an accessible name (`aria-label` / `loadingAriaLabel`).

- Submit while generating: `features/letter-form/letter-form.tsx`

**Invalid input** — Mirror validation errors for AT with `aria-invalid` on the field (alongside visible error UI).

- Additional details over limit: `features/letter-form/letter-form.tsx`
